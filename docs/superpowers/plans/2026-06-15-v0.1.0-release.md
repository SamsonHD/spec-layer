# v0.1.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a verified `release/v0.1.0` branch containing all current Spec Layer features and open-source hardening, then prepare a draft pull request and draft normal GitHub release.

**Architecture:** Preserve the two dirty worktrees as separate checkpoint commits before integration. Build the release candidate in a third isolated worktree from the complete feature branch, merge the cleanup branch, resolve overlaps in favor of current product behavior plus cleanup security constraints, and block GitHub delivery on the full local release gate.

**Tech Stack:** Git worktrees, npm workspaces, TypeScript, Next.js, React, Vitest, ESLint, GitHub Actions, GitHub CLI.

Run after resolution:

```bash

git status --short
```

Expected: no unmerged paths and no whitespace errors.
